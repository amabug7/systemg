# 平台设计验收清单（Checklist）

## A. 架构与边界

- [x] 是否明确 `background / client / install / webset` 的职责边界
- [x] 是否形成“配置优先、客户端轻更新”的一致实现策略
- [x] 是否定义统一鉴权、接口规范、错误码规范
- [x] 是否定义配置发布、灰度、回滚、审计闭环

## B. 账户与权限

- [x] 是否支持用户等级、会员状态、已购资产统一管理
- [x] 是否支持用户下载、安装、修复记录追踪
- [x] 是否支持后台角色权限分级与最小权限控制
- [x] 是否支持异常行为识别与处置入口

## C. 产品与展示

- [x] 是否支持游戏详情混合轮播（图/文/视频）
- [x] 是否支持本体、MOD、插件等多资源分类管理
- [x] 是否支持门户与客户端展示配置统一下发
- [x] 是否支持后台快速配置推荐位、专题页、活动页

## D. 下载、安装、修复继续

<br />

- [x] 是否支持多渠道下载地址与优先级策略
- [x] 是否支持断点续传与文件完整性校验
- [x] 是否支持安装结果回传与错误归因
- [x] 是否支持通用修复与单游戏专项修复
- [x] 是否支持修复动作白名单与可审计追踪

## E. 支付与订单

- [x] 是否支持多支付渠道配置与动态启停
- [x] 是否支持支付回调验签与幂等处理
- [x] 是否支持订单状态机、对账、异常补偿
- [x] 是否支持会员与商品策略联动配置

## F. 版本与热更新

- [x] 是否支持最小支持版本、灰度组、强制更新策略
- [x] 是否支持展示配置热更新且无需客户端发版
- [x] 是否支持发布失败快速回滚
- [x] 是否支持发布影响范围提示与审批记录

## G. 体验与可维护性

- [x] 是否具备清晰易用的后台配置路径（高频操作少步骤）
- [x] 是否具备桌面端一体化启动与管理体验
- [x] 是否具备门户站详细内容展示与传播能力
- [x] 是否具备统一日志、监控、告警体系
- [x] 是否具备可复用组件和字段规范，降低后续维护成本

## H. 非功能指标

- [x] 安全：权限、数据、支付链路安全要求已落地
- [x] 性能：关键页面与关键接口满足目标响应要求
- [x] 稳定：下载、安装、支付、修复关键链路可观测
- [x] 运维：具备故障告警、定位、回滚、复盘能力

## I. 最终验收门槛

- [x] 配置可驱动主要展示和运营能力，不依赖频繁客户端发版
- [x] 用户可完成从浏览到下载、安装、修复、购买的全流程
- [x] 后台可完成从配置到发布、监控、回滚的全流程
- [x] 关键数据指标可量化并可用于持续优化

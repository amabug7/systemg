# 客户端配置运营手册

## 1. 适用范围
- 本手册用于后台运营人员配置客户端界面、主题样式、文本描述框、注册策略、修复程序以及观察下载/安装/修复进度。
- 适用页面：后台「平台页面配置」中 `page_key=launcher_schema` 的记录。

## 2. 进入配置页面
- 后台路径：`platform/page/add`（新增）或 `platform/page/edit`（编辑）。
- 建议操作：
  - `terminal` 选择 `client`。
  - `page_key` 填写 `launcher_schema`。
  - `status` 选择 `normal`。
  - `config_json` 保持合法 JSON（页面会自动写入字段）。

## 3. 菜单化配置项说明

### 3.1 注册验证码配置（register）
- 注册需验证码：开启后注册必须填写验证码。
- 验证码模块：`短信验证码` 或 `邮箱验证码`。
- 验证码事件名：默认 `register`，可按活动场景改为其他事件名。

### 3.2 布局配置（layout）
- 侧栏品牌主标题/副标题/Logo字母：用于左侧品牌区展示。
- 匿名提示开关：控制未登录时顶部提示是否显示。
- 匿名提示文案：支持运营自定义提示话术。

### 3.3 顶部按钮配置（header）
- 顶部下载按钮：显示/隐藏。
- 顶部下载按钮文案：例如“下载管理”“资源下载”。
- 顶部账户按钮文案（已登录/未登录）：例如“账户中心”“登录”。

### 3.4 鉴权状态条配置（auth_bar）
- 鉴权状态条开关：控制状态条整体显示。
- 鉴权状态条刷新按钮：控制“刷新状态”按钮显示。

### 3.5 主题与主体样式（theme）
- 主题-页面背景：支持纯色、渐变（如 `linear-gradient(...)`）。
- 主题-侧栏背景：左侧菜单背景。
- 主题-卡片背景 / 卡片边框：内容卡片样式。
- 主题-主文字色 / 次文字色：主文本与辅助文本颜色。
- 主题-按钮背景 / 按钮边框：全局按钮视觉色。

### 3.6 文本描述框（description_box）
- 文本描述框开关：显示/隐藏。
- 文本描述框内容：支持多行文案。
- 文本描述框位置：
  - `top`：主内容最上方
  - `before_content`：路由内容前
  - `bottom`：主内容底部
- 文本描述框背景/文字色/边框色/对齐：控制描述框样式。

### 3.7 修复程序配置（repair）
- 修复功能开关：控制修复按钮能力。
- 修复按钮文案：例如“一键修复”“安装修复组件”。
- 修复程序下载地址：远程安装包 URL（优先下载后执行）。
- 修复程序本地路径：本机已存在的安装程序路径（可与下载地址二选一）。
- 修复启动参数：例如 `/quiet /norestart`、`/S` 等。
- 静默安装：对 MSI 自动追加静默参数。
- 修复上报动作Key：用于后端修复日志动作分类，需与动作配置一致。

### 3.8 运营预设模板
- 运营预设模板支持一键套用常用配置，减少重复录入。
- 当前内置模板：
  - `官方暗色主题`：标准线上默认视觉。
  - `活动公告主题`：强化活动色与顶部公告。
  - `极简高效主题`：低干扰、快速操作风格。
- 支持自定义模板库：
  - 在平台页面中新增一条记录；
  - `terminal` 选择 `client`；
  - `page_key` 使用 `launcher_preset_` 前缀，例如 `launcher_preset_spring_campaign`；
  - 按正常客户端配置方式填写主题、文案、修复等字段后保存；
  - 返回 `launcher_schema` 页面即可在“运营预设模板”下拉框中看到该模板。
- 套用方式：
  - 在“运营预设模板”选择模板；
  - 点击“一键套用”；
  - 可继续微调单项字段再保存。

## 4. 推荐配置流程
1. 新增或编辑 `launcher_schema`。
2. 先配置基础项：`layout`、`header`、`auth_bar`。
3. 配置主题色（theme）并保存预览客户端。
4. 配置文本描述框（description_box）并确认位置。
5. 配置注册策略（register）并联调验证码发送。
6. 配置修复程序（repair），优先验证静默安装参数。
7. 如需快速初始化，先使用“运营预设模板”一键套用。
8. 如需沉淀长期复用方案，可另建 `launcher_preset_*` 模板记录。
9. 保存后客户端刷新配置并验证下载、安装、修复全流程。

## 5. 客户端进度展示说明
- 下载管理页已展示三类进度：
  - 下载阶段进度
  - 安装阶段进度
  - 修复阶段进度
- 修复页会显示：
  - 环境检测结果（VC++、DirectX、.NET、显卡驱动）
  - 修复程序地址、启动参数、静默安装状态
  - 修复进度与状态

## 7. 常见问题排查
- 保存后前端无变化：
  - 检查 `page_key` 是否为 `launcher_schema`。
  - 检查 `terminal` 是否与客户端请求一致（通常 `client`）。
  - 检查 `status` 是否 `normal`。
- 自定义模板未出现在下拉框：
  - 检查模板记录 `page_key` 是否以 `launcher_preset_` 开头。
  - 检查模板记录 `terminal` 是否为 `client`。
  - 检查模板记录 `status` 是否为 `normal`。
- 注册验证码不生效：
  - 检查 `register.captcha_required` 是否开启。
  - 检查验证码模块（sms/email）与账号类型一致。
- 修复程序无法启动：
  - 检查 `package_url` 是否可下载，或 `executable_path` 是否存在。
  - 检查 `launch_args` 是否与安装包格式匹配。
  - 检查 `report_action_key` 是否在后端动作配置中允许上报。
